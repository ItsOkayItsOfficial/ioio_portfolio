Hal Bot

Overview

Hal is like iPhone's SIRI.

However, while SIRI is a Speech Interpretation and Recognition Interface, Hal is a Language Interpretation and Recognition Interface.

Hal is a console Node.js app that takes in parameters and gives you back data.

Scope

Utalizing Node.js, NPM, and CLI keywords we will pull data from the Twitter, Spotify, and OMDb API to dynamically display in console.

API Queried CLI Displayed Expected Outcome
Twitter Our last 20 tweets & when they were created.
Spotify User-specified track info: Name, Artist(s), Album, Preview URL.
OMDb User-specified movie info: Title (Year), Actors, IMDB/RT Rating, Country/Lang, Plot, Rating

There will be inputs for: getting keyword breakdowns, clearing logs, randomized outputs, & outputs tailored for failure of API queries or user-inputs

This is to say We will as well be logging this information in a text file for manipulation

  • .gitignore
              
              
This file specifies intentionally untracked files that Git should ignore.

              
However files already tracked by Git are not affected.
  • hal.js
              
              
Where most of our coding will take place

              
Here we will write the code we need to grab the data from keys.js, dependencies for Node.js from NPM, as well as basic scripting and logging for Hal.
  • keys.js
                  
              
Just a basic Javascript file where we will store the API access tokens and 'keys' for the Spotify and Twitter
                APIs to use

              
**Do Not** add this file to the .gitignore. This might be a good thing to do in the real world, but it for
                our purposes here we need it packaged with Hal
  • log.txt
                  
              
Our log.txt setup is where we will log all of the input, output, and interactions between the user, CLI, Hal,
                and the specific APIs
  • package-lock.json
                  
              
This is automatically generated for any operations where NPM modifies either the node_modules                tree, or package.json.

              
It describes the exact tree that was generated, such that subsequent installs are able to generate identical
                trees, regardless of intermediate dependency updates.
  • package.json
                  
              
This document is where we store the specific dependencies node.js will need to make Hal work.

              
It must be actual JSON, not just a JavaScript object literal.

              
For a complete document breakdown see this interactive guide
  • random.txt
                  
              
This is where we will store random returns to use.

              
Hal will take the text inside of random.txt and then use it to call one of Hal's commands.
  • README.md
                  
              
Our github README file for our repository is tell other people
                why, what they can do, and how they can use Hal Bot.

              
In this case we're using it to store the project instructions
  • node_modules
                  
              
Where our NPM "dependencies" packages are dropped when installed under their prefix.

              
This is folder is created automatically by node and populated with the dependencies we specified in our
                package.json
